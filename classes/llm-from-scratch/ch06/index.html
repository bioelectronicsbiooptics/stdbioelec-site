Introduction

Having coded the LLM architecture, pretrained it, and learned to import pretrained weights, the fruits of that labor are now reaped by fine-tuning the LLM on a specific target task: classifying text messages as "spam" or "not spam."

Figure 6.1 -- Three Main Stages of Coding an LLM
From the book (Raschka, 2025)
This chapter focuses on stage 3 (step 8): fine-tuning a pretrained LLM as a classifier. From the Foundation model, two paths diverge: classification fine-tuning (with class labels) and instruction fine-tuning (with instruction datasets).

6.1 Different Categories of Fine-Tuning

  • Instruction fine-tuning: Training on tasks with specific instructions to improve understanding and executing natural language prompts. Best for varied tasks; demands larger datasets and compute.
  • Classification fine-tuning: Training to recognize a specific set of class labels (e.g., "spam" / "not spam"). Highly specialized; requires less data and compute.
Figure 6.2 -- Instruction Fine-Tuning Scenarios
From the book (Raschka, 2025)
Top: Model determines if text is spam given instructions and a directive ("Answer with 'yes' or 'no'"). Bottom: Model translates English to German given an instruction.
Figure 6.3 -- Classification Fine-Tuning
From the book (Raschka, 2025)
A classification fine-tuned model does not require instructions alongside input. It can only respond with "spam" or "not spam" -- restricted to classes seen during training.
Choosing the Right Approach: Instruction fine-tuning provides flexibility across many tasks but demands more resources. Classification fine-tuning is ideal for precise categorization (sentiment analysis, spam detection) with less data and compute.

6.3 Creating Data Loaders

Text messages have varying lengths. Two options: truncate to shortest or pad to longest. Padding is chosen to preserve all content, using token ID 50256 (<|endoftext|>).

Figure 6.6 -- Input Text Preparation
From the book (Raschka, 2025)
Each text message is tokenized, then shorter sequences are padded with token ID 50256 to match the longest sequence length.
Listing 6.4: SpamDataset class
class SpamDataset(Dataset):
    def __init__(self, csv_file, tokenizer, max_length=None, pad_token_id=50256):
        self.data = pd.read_csv(csv_file)
        self.encoded_texts = [
            tokenizer.encode(text) for text in self.data["Text"]
        ]
        if max_length is None:
            self.max_length = self._longest_encoded_length()
        else:
            self.max_length = max_length
            self.encoded_texts = [et[:self.max_length] for et in self.encoded_texts]
        self.encoded_texts = [
            et + [pad_token_id] * (self.max_length - len(et))
            for et in self.encoded_texts
        ]

    def __getitem__(self, index):
        encoded = self.encoded_texts[index]
        label = self.data.iloc[index]["Label"]
        return (torch.tensor(encoded, dtype=torch.long),
                torch.tensor(label, dtype=torch.long))

    def __len__(self):
        return len(self.data)

Longest sequence: 120 tokens. Batches: 130 training, 19 validation, 38 test (batch size 8).

6.5 Adding a Classification Head

Replace the output layer (768 -> 50,257) with a smaller classification layer (768 -> 2 classes).

Figure 6.9 -- Adapting GPT for Spam Classification
From the book (Raschka, 2025)
The original linear output layer mapping 768 hidden units to 50,257 tokens is replaced with a new layer mapping 768 units to just 2 classes ("spam" and "not spam").
# Freeze all layers
for param in model.parameters():
    param.requires_grad = False

# Replace output layer
torch.manual_seed(123)
num_classes = 2
model.out_head = torch.nn.Linear(
    in_features=BASE_CONFIG["emb_dim"], out_features=num_classes
)

# Make last transformer block and final LayerNorm trainable
for param in model.trf_blocks[-1].parameters():
    param.requires_grad = True
for param in model.final_norm.parameters():
    param.requires_grad = True
Figure 6.12 -- Why Focus on the Last Token?
From the book (Raschka, 2025)
Due to causal attention masking, the last token is the only one with attention scores to all preceding tokens. It accumulates the most information, making it ideal for classification.

Output shape changes from [1, 4, 50257] to [1, 4, 2]. We use only the last row (outputs[:, -1, :]) for classification.

6.6 Calculating the Classification Loss and Accuracy

Listing 6.8: Classification accuracy
def calc_accuracy_loader(data_loader, model, device, num_batches=None):
    model.eval()
    correct_predictions, num_examples = 0, 0
    if num_batches is None:
        num_batches = len(data_loader)
    else:
        num_batches = min(num_batches, len(data_loader))
    for i, (input_batch, target_batch) in enumerate(data_loader):
        if i < num_batches:
            input_batch = input_batch.to(device)
            target_batch = target_batch.to(device)
            with torch.no_grad():
                logits = model(input_batch)[:, -1, :]
            predicted_labels = torch.argmax(logits, dim=-1)
            num_examples += predicted_labels.shape[0]
            correct_predictions += (predicted_labels == target_batch).sum().item()
        else:
            break
    return correct_predictions / num_examples

6.8 Using the LLM as a Spam Classifier

Listing 6.12: Classifying new texts
def classify_review(text, model, tokenizer, device, max_length=None,
                    pad_token_id=50256):
    model.eval()
    input_ids = tokenizer.encode(text)
    supported_context_length = model.pos_emb.weight.shape[0]
    input_ids = input_ids[:min(max_length, supported_context_length)]
    input_ids += [pad_token_id] * (max_length - len(input_ids))
    input_tensor = torch.tensor(input_ids, device=device).unsqueeze(0)

    with torch.no_grad():
        logits = model(input_tensor)[:, -1, :]
    predicted_label = torch.argmax(logits, dim=-1).item()
    return "spam" if predicted_label == 1 else "not spam"

Summary

  • Two main strategies: classification fine-tuning and instruction fine-tuning
  • Classification fine-tuning replaces the output layer with a small classification layer (2 output nodes instead of 50,256)
  • The model is trained to output correct class labels instead of next tokens
  • Model input remains text converted to token IDs, same as pretraining
  • Pretrained model serves as a base model for fine-tuning
  • Evaluation uses classification accuracy (fraction of correct predictions)
  • Fine-tuning uses the same cross entropy loss as pretraining
  • The last output token is used for classification due to causal attention giving it access to all previous tokens
